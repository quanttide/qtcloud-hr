# 薪资核算

## 故事 1：计时工资试算

### 用户故事

作为核算方，我想要输入员工的工时、费率和扣款数据后即时得到试算结果，以便在正式提交前确认各项金额无误。

### 验收标准

**场景**：正常试算
- **假设**：核算方已获得员工的基础工时、小时费率、加班工时和扣款金额
- **当**：核算方输入基础工时 160、小时费率 25、加班工时 10、扣除金额 200 并发起试算
- **那么**：系统返回五项明细——基础薪资 4000、加班薪资 375、绩效奖金 400、扣除金额 200、净薪资 4575

**场景**：扣除金额超过应发总额
- **假设**：核算方输入的各项金额中扣除金额过大
- **当**：核算方输入基础工时 160、小时费率 25、扣除金额 5000 并发起试算
- **那么**：系统返回净薪资为 0，避免出现负数薪资

**场景**：输入不合法的参数
- **假设**：核算方输入了负数
- **当**：核算方输入基础工时 -10
- **那么**：系统拒绝计算并提示参数不合法

### 业务规则

- 基础薪资 = 基础工时 × 小时费率
- 加班薪资 = 加班工时 × 小时费率 × 1.5
- 绩效奖金 = 基础薪资 × 10%
- 净薪资 = 基础薪资 + 加班薪资 + 绩效奖金 - 扣除金额
- 净薪资最低为 0（保底机制）
- 工时、费率、扣款均不可为负数
- 所有金额结果保留两位小数
